1. PROJECT OVERVIEW
================================================================================
We are building a multi-user, role-based Fuel Requisition application. 
It replaces a manual process with a digital workflow: 
Request -> Supervisor Approval -> PO Generation -> Receiving -> Invoice Reconciliation.

TECH STACK:
- Language: Python 3.9+
- UI Framework: Streamlit
- Database: SQLite (Local file-based). MUST use WAL mode for concurrency.
- Security: bcrypt for password hashing (Never store plain text passwords).
- PDF Generation: jinja2 + xhtml2pdf (HTML-to-PDF approach).
- Network: Run on 0.0.0.0 for LAN and ZeroTier VPN visibility.

3. ARCHITECTURE & RULES
================================================================================

DATABASE RULES (FileMaker-like Logic):
1. Soft Deletes: Do not DELETE records from Master Data (Vendors/Cars/Users). 
   Use an 'is_active' BOOLEAN column (1=Active, 0=Inactive).
2. Concurrency: Enable Write-Ahead Logging (WAL) in 'src/database.py' to handle 
   multiple users simultaneously.
3. Foreign Keys: Store IDs, not names, in transaction tables 
   (e.g., store 'vendor_id', not 'vendor_name').

USER ROLES & PERMISSIONS:
- User: Create Request, View Own History.
- Supervisor: View Pending Requests, Approve/Reject.
- Purchaser: View Approved Requests, Generate PO (PDF), Mark PO as "Received/Delivered".
- Finance: View Received items, Reconcile with Vendor Invoice (Billing), 
  Manage Master Data, Manage Users.
